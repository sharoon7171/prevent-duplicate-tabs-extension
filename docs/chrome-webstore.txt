Prevent Duplicate Tabs – Chrome Extension | Stop Opening the Same Tab Twice

Stop duplicate tabs automatically. This free Chrome extension detects when you open the same webpage again and closes or switches tabs so you keep a cleaner browser and fewer tabs.

WHAT DOES THIS EXTENSION DO?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Prevent Duplicate Tabs watches when you open a new tab. If the URL matches one you already have open, it acts immediately:

1. Detects duplicate tabs – Compares the new tab’s URL with your open tabs (across all windows or only the current window—you choose).
2. Applies your chosen action – Closes the duplicate and optionally switches you to the right tab (and focuses the correct window).
3. Respects your rules – You choose what counts as a duplicate and where duplicates are allowed.

You stay in control: enable or disable the extension anytime, and set global and per-site rules. You can also choose whether to prevent duplicates across all windows or only within the current window.

HOW TO HANDLE DUPLICATES (YOU CHOOSE)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

When a duplicate is detected, you can set one of these actions (globally or per website):

• Close the new duplicate tab and stay on your current tab – You keep working where you are; the new tab is closed.
• Close the old duplicate and stay on your current tab – The older tab is closed; you stay on the one you’re on.
• Close the new duplicate and switch to the existing tab – The new tab is closed and you’re taken to the tab that was already open.
• Close the old duplicate and switch to the new tab – The old tab is closed and you’re taken to the new one.

You can set a default for all sites and override it for specific domains (e.g. different behavior for Gmail vs. docs).

When you open a tab with a URL that is already open, the extension detects it and closes or switches tabs according to your chosen setting (e.g. close the new tab, or close the old one and switch). PERMISSION JUSTIFICATIONS (max 1000 characters each)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

tabs:
The extension needs the "tabs" permission to fulfill its single purpose of preventing duplicate tabs. It uses chrome.tabs to: (1) Listen for new tabs (onCreated) and tab URL changes (onUpdated) so it can detect when a user opens a URL that is already open. (2) Query all open tabs (chrome.tabs.query) to compare the new tab’s URL with existing tabs and identify duplicates. (3) Get the currently active tab so the popup can show settings for “this page” or “this domain” and display the correct tab count. (4) Close duplicate tabs (chrome.tabs.remove) and switch the active tab (chrome.tabs.update) when the user’s chosen action is “close and switch.” Without the tabs permission, the extension cannot read tab URLs to detect duplicates or close/switch tabs; no other APIs provide this capability.

storage:
The extension needs the "storage" permission to fulfill its single purpose of preventing duplicate tabs. It uses chrome.storage.sync to: (1) Save the user’s on/off preference so the extension can be enabled or disabled. (2) Save the user’s chosen duplicate action (e.g. close new tab, close old tab, switch to existing), “prevent duplicates in” scope (all windows or current window only), and “ignore URL parameters” setting so the correct behavior is applied. (3) Save the user’s exception lists (specific URLs and domains where duplicates are allowed) and site-specific rules so duplicate detection respects the user’s choices. (4) Store a count of “duplicates prevented” shown in the popup. Without storage, the extension could not remember any settings or exceptions and would not function as intended. No data is sent to external servers; storage is synced by Chrome across the user’s device.

windows:
The extension needs the "windows" permission to fulfill its single purpose of preventing duplicate tabs when the user’s chosen action involves another window. It uses chrome.windows to: (1) Focus the correct window when the user chooses “switch to existing tab” or “switch to new tab” and that tab is in a different window, so the user is taken to the right window. (2) Close a window that is left empty after the extension removes a duplicate tab (e.g. when a new window’s only tab was a duplicate). Without the windows permission, the extension could not focus the target window or clean up empty windows; tab switching would be less effective and empty windows would remain. No window data is stored or sent elsewhere.
