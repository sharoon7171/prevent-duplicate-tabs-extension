The extension needs the "tabs" permission to fulfill its single purpose of preventing duplicate tabs. It uses chrome.tabs to: (1) Listen for new tabs (onCreated) and tab URL changes (onUpdated) so it can detect when a user opens a URL that is already open. (2) Query all open tabs (chrome.tabs.query) to compare the new tab’s URL with existing tabs and identify duplicates. (3) Get the currently active tab so the popup can show settings for “this page” or “this domain” and display the correct tab count. (4) Close duplicate tabs (chrome.tabs.remove) and switch the active tab (chrome.tabs.update) when the user’s chosen action is “close and switch.” (5) Open a tab from the context menu when the user chooses “Open in new tab (allow duplicate).” Without the tabs permission, the extension cannot read tab URLs to detect duplicates or close/switch tabs; no other APIs provide this capability.
